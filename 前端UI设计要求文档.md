# 宝宝喂养日志 (BabyLog+) 前端UI设计要求文档

## 文档信息

- **项目名称**: 宝宝喂养日志 (BabyLog+)
- **目标平台**: 微信小程序（主要）+ 多端支持（H5等）
- **UI框架**: WotUI (基于 NutUI 组件库)
- **技术栈**: Vue 3 + TypeScript + Vite
- **设计风格**: 现代简约、温馨友好、符合母婴产品特性
- **文档版本**: v1.0
- **创建日期**: 2025-11-07

---

## 1. 设计原则与风格指导

### 1.1 核心设计原则

- **用户中心**: 以宝宝和家长的使用体验为核心
- **简约清晰**: 界面简洁，信息层次分明
- **温馨友好**: 色彩温暖，视觉亲和力强
- **高效便捷**: 减少操作步骤，提升记录效率
- **一致统一**: 统一的视觉语言和交互规范

### 1.2 视觉风格

**色彩系统**:
- 主色调: 温暖的粉蓝色 (#FFB6C1, #87CEEB) - 代表关爱与成长
- 辅助色: 柔和的绿色 (#98FB98) - 代表健康与活力
- 强调色: 温暖的橙色 (#FFA500) - 用于重要操作和提醒
- 中性色: 温和的灰色系 (#F5F5F5, #E0E0E0, #9E9E9E)
- 文字色: 深灰 (#333333) 主要文字，中灰 (#666666) 次要文字

**字体规范**:
- 主字体: 系统默认字体（苹方/思源黑体）
- 标题: 16-20px，粗体
- 正文: 14-16px，常规
- 辅助文字: 12px，常规
- 数字/统计: 使用等宽字体以保证对齐

**圆角与阴影**:
- 卡片圆角: 12rpx
- 按钮圆角: 8rpx
- 输入框圆角: 8rpx
- 阴影: 轻微投影，0 2rpx 8rpx rgba(0,0,0,0.08)

### 1.3 布局规范

- **安全区域**: 适配各种机型的安全区域
- **页面边距**: 左右边距 32rpx，上下边距 24rpx
- **组件间距**: 16rpx (紧凑)、24rpx (常规)、32rpx (宽松)
- **栅格系统**: 12列栅格系统，列间距 16rpx
- **响应式设计**: 基于 750rpx 基准进行响应式适配

**核心要素**:

- 奶量输入支持小数

**UI组件**:
- NutUI Tabs (喂养类型)
- NutUI ButtonGroup (左右胸选择)
- NutUI InputNumber (数量输入)
- NutUI Picker (食物类型)
- NutUI Uploader (照片)
- NutUI Textarea (备注)

## 4. 交互设计规范

### 4.1 手势交互

**点击**:
- 响应时间: < 100ms
- 反馈: 按钮按下态（透明度 0.8）
- 区域: 最小 44rpx × 44rpx

**滑动**:
- 最小滑动距离: 50rpx
- 方向: 左/右/上/下
- 应用: 滑动删除、轮播、时间轴滑动

**长按**:
- 触发时间: 500ms
- 反馈: 显示操作菜单或进入编辑模式

**下拉刷新**:
- 触发距离: 100rpx
- 动画: 旋转刷新图标

**上拉加载**:
- 触发距离: 50rpx
- 加载状态: "加载更多" + 旋转图标

### 4.2 动画效果

**页面切换**:
- 方式: 左右滑动（300ms）
- 缓动函数: ease-in-out

**弹窗出现**:
- 方式: 缩放 + 淡入（200ms）
- 缓动函数: ease-out

**数据更新**:
- 方式: 淡入（150ms）
- 列表项: 依次淡入

**按钮点击**:
- 方式: 轻微缩放（0.95，150ms）
- 缓动函数: ease-out

### 4.3 状态设计

**加载状态**:
- 骨架屏: 灰色占位
- 加载动画: 旋转图标
- 文字: "加载中..."

**空状态**:
- 插画: 温馨的主题插画
- 提示文字: 引导性文案
- 操作按钮: "去添加" 或 "刷新"

**错误状态**:
- 图标: 感叹号或错误标志
- 提示文字: 友好说明 + 解决建议
- 操作按钮: "重试" 或 "返回"

## 10. 开发注意事项

### 10.1 组件使用
- 优先使用 WotUI (NutUI) 组件
- 自定义组件需继承基础组件样式
- 保持组件风格一致性
- 复用率低的组件可原生实现

- 新页面需在 pages.json 中注册
- 路径格式: "pages/页面名/index"
- 避免路径层级过深（最多 3 层）

### 10.3 状态管理
- 使用 Vue 3 reactive
- 避免全局状态过多
- 页面级状态使用 provide/inject
- 大数据量使用深拷贝

### 10.4 网络请求
- 统一使用 request 工具
- 错误处理使用 Toast
- 关键数据本地缓存
- 离线状态友好提示
